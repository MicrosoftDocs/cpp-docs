# #import Attributes (C++)
Provides links to attributes used with the #import directive.  
  
 **Microsoft Specific**  
  
 The following attributes are available to the #import directive.  
  
|Attribute|Description|  
|---------------|-----------------|  
|[auto_rename](../c/auto_rename.md)|Renames C++ reserved words by appending two underscores (__) to the variable name to resolve potential name conflicts.|  
|[auto_search](../c/auto_search.md)|Specifies that, when a type library is referenced with #import and itself references another type library, the compiler can do an implicit #import for the other type library.|  
|[embedded_idl](../c/embedded_idl.md)|Specifies that the type library is written to the .tlh file with the attribute-generated code preserved.|  
|[exclude](../c/exclude--sharpimport-.md)|Excludes items from the type library header files being generated.|  
|[high_method_prefix](../c/high_method_prefix.md)|Specifies a prefix to be used in naming high-level properties and methods.|  
|[high_property_prefixes](../c/high_property_prefixes.md)|Specifies alternate prefixes for three property methods.|  
|[implementation_only](../c/implementation_only.md)|Suppresses the generation of the .tlh header file (the primary header file).|  
|[include()](../c/include--.md)|Disables automatic exclusion.|  
|[inject_statement](../c/inject_statement.md)|Inserts its argument as source text into the type-library header.|  
|[named_guids](../c/named_guids.md)|Tells the compiler to define and initialize GUID variables in old style, of the form **LIBID_MyLib**, **CLSID_MyCoClass**, **IID_MyInterface**, and **DIID_MyDispInterface**.|  
|[no_auto_exclude](../c/no_auto_exclude.md)|Disables automatic exclusion.|  
|[no_dual_interfaces](../c/no_dual_interfaces.md)|Changes the way the compiler generates wrapper functions for dual interface methods.|  
|[no_implementation](../c/no_implementation.md)|Suppresses the generation of the .tli header, which contains the implementations of the wrapper member functions.|  
|[no_namespace](../c/no_namespace.md)|Specifies that the namespace name is not generated by the compiler.|  
|[no_registry](../c/no_registry.md)|Tells the compiler not to search the registry for type libraries.|  
|[no_search_namespace](../c/no_search_namespace.md)|Has the same functionality as the [no_namespace](../c/no_namespace.md) attribute but is used on type libraries that you use the #import directive with the [auto_search](../c/auto_search.md) attribute.|  
|[no_smart_pointers](../c/no_smart_pointers.md)|Suppresses the creation of smart pointers for all interfaces in the type library.|  
|[raw_dispinterfaces](../c/raw_dispinterfaces.md)|Tells the compiler to generate low-level wrapper functions for dispinterface methods and properties that call **IDispatch::Invoke** and return the `HRESULT` error code.|  
|[raw_interfaces_only](../c/raw_interfaces_only.md)|Suppresses the generation of error-handling wrapper functions and [property](../cpp/property--c---.md) declarations that use those wrapper functions.|  
|[raw_method_prefix](../c/raw_method_prefix.md)|Specifies a different prefix to avoid name collisions.|  
|[raw_native_types](../c/raw_native_types.md)|Disables the use of COM support classes in the high-level wrapper functions and forces the use of low-level data types instead.|  
|[raw_property_prefixes](../c/raw_property_prefixes.md)|Specifies alternate prefixes for three property methods.|  
|[rename](../c/rename--sharpimport-.md)|Works around name collision problems.|  
|[rename_namespace](../c/rename_namespace.md)|Renames the namespace that contains the contents of the type library.|  
|[rename_search_namespace](../c/rename_search_namespace.md)|Has the same functionality as the [rename_namespace](../c/rename_namespace.md) attribute but is used on type libraries that you use the #import directive with the [auto_search](../c/auto_search.md) attribute.|  
|[tlbid](../c/tlbid.md)|Allows for loading libraries other than the primary type library.|  
  
 **END Microsoft Specific**
